Colour

Our design system uses colour in a range of meaningful and intentional ways to create patterns and visual cues. We provide the following colour palettes:

• main palette
• contrast main palette
• dark palette
• contrast dark palette
• notification palette

These palettes work together to support a variety of uses cases and provide a consistent look for action or link colours and focus states regardless of the palette combination.

Our colour palettes have also been designed to be implemented in a themeable manner. The universal colour variables are determined by common roles and usage; they are not based singularly on a colour value (ie unique hex code). The same colour value may be assigned to multiple variables in a theme's palette when the values have distinctly different roles.

Font families (System fonts)

We use system fonts as they are used by the operating system for displaying text used in menus, icons and errors. Performance may be boosted when the system defaults to a system font because the browser doesn't have to download any font files. It's using one it already had.

System fonts have come a long way in recent times. Manufacturers are now shipping some very elegant fonts as default in their operating systems.

During research we met many Australians who live in rural and remote areas where Internet access is limited but government services are still required. Using system fonts makes the load times a whole lot faster for such users with only a slight compromise on aesthetics.

Spacing

A consistent unit of measurement is required to avoid layout inconsistencies between pages.

Our layout spacing is based on an 8pt grid system. We use a ‘soft’ 8-pt grid system to position individual elements relative to each other instead of to an actual grid (a ‘hard grid’). This means we use increments of 8 to size and space out elements on a page. So any defined height or width, margin or padding will be an increment of 8.

By using an even number like 8 to size and space elements, we are able to also scale for a wide variety of devices easily and consistently.

In our design system we specify 1unit as 16px or 1rem.

We use column grids to align content horizontally, and the 8pt baseline grid to vertically align bodies of text.

Screen readers only

mixin: AU-sronly

Use the AU-sronly mixin for content which should be visually hidden, but remain accessible to assistive technologies such as screen readers. This can be useful in situations where additional visual information or cues (such as meaning denoted through the use of color) need to also be conveyed to non-visual users.

Copy code
@mixin AU-sronly() {
  position: absolute;
  width: 1px;
  height: 1px;
  padding: 0;
  margin: -1px;
  overflow: hidden;
  clip: rect(0,0,0,0);
  border: 0;
}

 

 

Focus styles

mixin: AU-focus
mixin: AU-outline

Use the AU-focus mixin to add a focus state to any element that is consistent with other elements in the design system.

When used correctly this mixin ensures the focus state is visible in both our light and dark colour themes.

HTML   React
Copy code
/**
 * AU-outline - Create outline based on the theme the user is using.
 *
 * @param  {keywords} $theme - `dark` or default ( `light` )
 *
 * @return {string}          - The code
 */
@mixin AU-outline( $theme: 'light' ) {
	@if $theme == 'dark' {
		outline: 3px solid $AU-colordark-Focus;
	}
	@else {
		outline: 3px solid $AU-color-Focus;
	}
}


/**
 * AU-focus - Add the outline to focus
 */
@mixin AU-focus( $theme: 'light' ) {
	&:focus {
		@include AU-outline( $theme );
	}

	// Remove Modzilla inner HTML dotted line. github.com/necolas/normalize.css/blob/master/normalize.css#L285
	&::-moz-focus-inner {
		border: 0;
	}
}
